Overview

This is for educational/research purposes only

Hex Factor is based on the Settlers of Catan, a turn-based board game I really play too much. Each player strives to be the first to gain 10 points. Points are based on the number of settlements (1x), cities (2x), if you have the largest army [of 3 or more knights] (+2), and if you have the longest road [of 5 or more segments] (+2). Points are gained through the building of roads, settlements and cities using resource cards that are gained when the die roll matches the number of the tile which a player has an existing adjacent settlement or city. Development cards can also be purchased with a combination of resources, and can alter gameplay in a variety of ways.

Background

I originally started implementing the game controller with a Swing-based client, but ultimately wanted to have a rich web client. Google App Engine makes hosting the game very easy.

Goals

  • To be able to play at any time, anywhere, even if I left my ragged board at home
  • To gather and analyze gameplay data (which hopefully translates to me winning more)

Technical Details

  • The client is written with GWT
  • The board is rendered using SVG and CSS
  • Web Method calls are made using GWT-RPC
  • Game events are "pushed" to each client using Google App Engine Channels

Current Limitations and Known Issues

  • You must have a Google Account or OpenID account to play
  • All invitees must join the game before it can start
  • It is best if you already know how to play and what resources are required to build anything
  • There is no robber (no 7s can be rolled)
  • There are only 2 development card types implemented
  • Only 50 simultaneous user channels are allowed, so no more than 12 games of 4 or 16 games of 3, or somewhere in between, will be served (so I can stay within the Google App Engine "free" price tier)
  • Unanswered game invitations will expire after 60 minutes
  • Inactive games will expire after 60 minutes
  • There is no way to re-enter a game in progress, so don't close your browser
  • Game Art is not my forte

Future Development Goals

Other than resolving the current limitations and known issues, I'd ultimately like to have the following

  • Further decouple the board rendering model by having the plain board model and then transform it to SVG using an XSL template
  • Integrate a rules engine for defining your own rules when creating a game
  • Use Open Auth for player identification
  • Track every turn in datastore
